Вот структурированный и профессиональный README.md для твоего Go-сервиса (geocash-analytics). Он составлен на основе твоего кода main.go и учитывает автоматическую инициализацию базы данных, интеграцию с gRPC и структуру хендлеров.

🏧 GeoCash Analytics Backend (Go)
Центральное звено системы аналитики банкоматов. Сервис отвечает за хранение геоданных в PostGIS, обработку бизнес-логики и агрегацию данных из внешнего Python AI-сервиса.

🚀 Быстрый старт
1. Переменные окружения
Сервис настраивается через переменные окружения. Создайте .env или передайте их при запуске:

DB_HOST: Хост базы данных PostgreSQL (по умолчанию localhost).

AI_SERVICE_ADDR: Адрес Python-сервиса скоринга (по умолчанию localhost:50051).

2. Запуск
Bash
go run cmd/app/main.go
После запуска сервер будет доступен по адресу: http://localhost:8080

🛠 Архитектура и компоненты
📡 API Эндпоинты
GET /: Проверка работоспособности (Health Check).

GET /api/dashboard: Основной агрегатор данных. Возвращает:

forte: Список собственных банкоматов с их статусами эффективности и жалобами клиентов.

competitors: Точки конкурентов (Kaspi, Halyk и др.).

traffic: GeoJSON сегменты дорожной сети с данными о проходимости.

recommendations: Оптимальные точки для новых банкоматов, полученные от AI-сервиса.

🧠 Слой логики (Handlers & Services)
В проекте реализовано четкое разделение ответственности:

Handlers (internal/dashboard/handler.go): Принимают запросы, обеспечивают поддержку CORS и вызывают сервисы.

Services (internal/dashboard/service.go): Содержат бизнес-логику и оркеструют данные.

Repository (internal/platform/postgres): Прямое взаимодействие с SQL-запросами и PostGIS.

🗄 Работа с данными
Автоматическая инициализация (Seed)
При каждом запуске функция ensureSchema выполняет следующие действия:

Очистка: Удаляет старые таблицы для обеспечения консистентности.

Schema: Создает таблицу terminals с поддержкой гео-координат.

Seeding: Наполняет базу ~380 точками:

Реальные якоря: Ключевые точки ForteBank (Москва, MEGA, Хан Шатыр).

Проблемные зоны: Точки с искусственно сгенерированными жалобами (грязные экраны, поломки).

Массовка: 300+ точек конкурентов для анализа покрытия.

Интеграция трафика
Сервис автоматически ищет файл traffic_data.csv в корне. Если файл найден, запускается фоновый процесс (goroutine) для загрузки сегментов трафика в базу данных через TrafficIntegrator.

🤖 gRPC Интеграция с AI Service
Бэкенд выступает в роли gRPC-клиента для Python-сервиса.

Логика работы: При запросе дашборда сервис запрашивает рекомендации для заданных координат (по умолчанию центр Астаны).

Отказоустойчивость: Если AI-сервис недоступен, бэкенд продолжит работу, отдавая пустой список рекомендаций и записывая предупреждение в лог: ⚠️ WARNING: AI Service недоступен.
